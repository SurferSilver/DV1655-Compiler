C+- grammar
The grammar used in this assignment is shown in Listing 1.
In the C+- language:
    • Statements end with NEWLINE (no semicolons).

    • Assignment uses :=.

    • Arrays are one-dimensional and only supported for primitive types (int, float, boolean).
    
    • The entry point of the program is main() : int.
    
    • Variables are immutable by default. Only variables declared volatile may be assigned after
    initialization.
    
    • No shadowing: a name may not be re-declared in any inner scope (program/class/method/block),
    and you may not redeclare globals inside classes or methods.
    
    • No nested methods: methods may only appear inside classes (not inside other methods or blocks).
    
    • All identifiers must be declared before use (variables, methods, classes), and local variables must
    be declared before they are referenced.
    
    • Type rules: operators require compatible operand types (e.g., arithmetic on numbers, logical
    ops on booleans), and assignment/return types must match the declared type (implicit numeric
    conversions allowed).
    
    • Control-flow rules: break and continue are only allowed inside loops; and a return must match
    the method return type.
    
    • .length is only valid on arrays
